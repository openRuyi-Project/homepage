We are pleased to announce the May 2026 release of openRuyi.

This month, openRuyi continued to expand the desktop experience, core system stack, and package ecosystem for RISC-V platforms.

{/* truncate */}

## Highlights

* **KDE desktop environment**: We have introduced a KDE-based desktop experience for openRuyi. This package now includes essential Plasma components and basic KDE applications, providing users with a more complete graphical desktop environment for daily use and development.

* **Modern web browser**: We added Firefox to openRuyi. This adds a modern browser option to the desktop image and improves the usability of openRuyi as a daily workstation.

* **Kernel and platform updates**: We updated the mainline kernel from `7.0.2` to `7.0.10` and the LTS kernel from `6.18.25` to `6.18.33`. We also refreshed `linux-tools` and the kernel headers, raised `NR_CPUS` to `256` to better support high-core-count RISC-V server platforms, and tuned memory management options to improve process and memory performance.

* **Toolchain updates**: We updated GCC from `15` to `16` and binutils from `2.45` to `2.46`. We also added LLVM version `22`, which improves the availability of the compiler for package maintainers and developers.

* **Storage, virtualization, and server stack improvements**: We updated Ceph to `20.2.1` and applied patches to resolve full test suite issues. We also added `libvirt`, `corosync`, `pacemaker`, and related packages, strengthening openRuyi’s virtualization, clustering, and server-side capabilities.

## Security Updates

We delivered multiple security updates this month.

* `curl` was updated to 8.20.0, including fixes for multiple newly disclosed vulnerabilities.

* `expat` was updated to 2.8.1, addressing [CVE-2026-45186], a denial-of-service issue in crafted XML input handling.

* `python-django` was updated to 5.2.13, addressing several Django security issues, including [CVE-2026-3902], [CVE-2026-4277], [CVE-2026-4292], [CVE-2026-33033], and [CVE-2026-33034].

* `libarchive` was updated to 3.8.7, bringing security and bug fixes for several archive parsing paths.

* `freerdp` was updated to 3.26.0, including fixes for high-severity CVEs.

* `nginx` was updated to 1.31.1, including fixes for [CVE-2026-9256], [CVE-2026-42926], [CVE-2026-42945], [CVE-2026-42946], [CVE-2026-40460], and [CVE-2026-40701].

* `rsync` was updated to 3.4.3, including fixes for [CVE-2026-43617], [CVE-2026-43618], [CVE-2026-43619], [CVE-2026-43620], [CVE-2026-45232], and [CVE-2026-29518].

* `unbound` was updated to 1.25.1, including fixes for [CVE-2026-33278], [CVE-2026-42944], [CVE-2026-42959], [CVE-2026-32792], [CVE-2026-40622], [CVE-2026-41292], [CVE-2026-42534], [CVE-2026-42923], [CVE-2026-42960], [CVE-2026-44390], and [CVE-2026-44608].

* `postgresql` was updated to 18.4, including fixes for [CVE-2026-6476], [CVE-2026-6638], [CVE-2026-6637], [CVE-2026-6473], [CVE-2026-6474], [CVE-2026-6479], [CVE-2026-6475], [CVE-2026-6478], [CVE-2026-6477], [CVE-2026-6575], and [CVE-2026-6472].

* The Linux kernel was updated with fixes for several notable CVEs, including [CVE-2026-31431], [CVE-2026-43284], [CVE-2026-43494], [CVE-2026-43500], [CVE-2026-43503], and [CVE-2026-46333].

* We also updated components such as `gnutls`, `nss`, `libtasn1`, `openldap`, `nghttp2`, `ruby`, and `vim`, bringing in related security fixes and improvements.

We continue to provide four editions for different scenarios: **Server**, **Container**, **Desktop**, and **Zero (BSP validation)**. You can download it from the [openRuyi release site](https://releases.openruyi.cn/creek/2026.05/rva23/).
